## 1. 项目概述

### 1.1 目标

将《数据结构习题集》PDF（40 页扫描图片，9 个章节，约 511 道选择题 + 填空题）制作成一个**单 HTML 文件刷题 App**，通过 GitHub Pages 部署，支持 PWA 安装到手机/桌面主屏幕，获得类原生 App 体验。

### 1.2 核心约束

- **单文件**：所有 HTML/CSS/JS/数据/图标 全部内嵌在一个 `index.html` 中
- **零依赖**：不依赖任何第三方库、CDN、框架、后端服务
- **离线可用**：PWA 安装后可完全离线刷题
- **GitHub Pages 部署**：推送即上线

### 1.3 题库概况

| 章节 | 单选 | 填空 | 合计 |
|------|------|------|------|
| 第1章 绪论 | 18 | 14 | 32 |
| 第2章 线性表 | 38 | 36 | 74 |
| 第3章 栈和队列 | 35 | 27 | 62 |
| 第4章 串 | 5 | 4 | 9 |
| 第5章 数组和广义表 | 20 | 18 | 38 |
| 第6章 树和二叉树 | 50 | 48 | 98 |
| 第7章 图 | 43 | 30 | 73 |
| 第8章 查找 | 36 | 26 | 62 |
| 第9章 排序 | 39 | 24 | 63 |
| **合计** | **284** | **227** | **511** |

## 2. 技术方案

### 2.1 整体架构

```
index.html (单文件)
├── <head>
│   ├── Meta 标签 (charset, viewport, theme-color, PWA)
│   ├── 内联 <style> (全部 CSS)
│   ├── PWA manifest (内联 data URI)
│   └── Icons (base64 SVG/PNG，Canvas 运行时生成)
├── <body>
│   ├── 封面页 (<section id="cover">)
│   │   ├── CSS 二叉树动画图标
│   │   ├── 标题 & 统计
│   │   └── 开始按钮 + 继续上次入口
│   ├── 答题页 (<section id="quiz">)
│   │   ├── 顶部工具栏 (章节/模式/题数选择器)
│   │   ├── 题目卡片区
│   │   ├── 选项/填空输入区
│   │   ├── 反馈 & 解析区
│   │   └── 底部进度条
│   └── 结算页 (<section id="result">)
│       ├── 圆形正确率图示
│       ├── 章节得分明细
│       └── 错题回顾列表
└── <script>
    ├── 题库数据 (questions JSON, ~300KB)
    ├── 答题引擎
    ├── UI 渲染 & 动画
    ├── localStorage 读写
    └── PWA service worker 注册 (内联)
```

### 2.2 技术选型

| 维度 | 选择 |
|------|------|
| 页面切换 | CSS `transform` + `opacity` 过渡动画 |
| 图标绘制 | HTML5 Canvas → toDataURL() → base64 |
| 数据存储 | localStorage（进度、统计、错题记录） |
| Service Worker | 内联 Blob URL 注册，缓存自身 |
| 响应式 | CSS Media Queries，移动端 ≤768px 自适应 |

### 3.7 动画规范

| 场景 | 动画 | 时长 | 缓动 |
|------|------|------|------|
| 封面→答题 | `opacity` 0→1 + `translateY` 20px→0 | 400ms | ease-out |
| 新题切入 | `translateX` 30px→0 + `opacity` 0→1 | 300ms | ease-out |
| 正确反馈 | `box-shadow` 绿色光晕脉冲 | 500ms | ease-in-out |
| 错误反馈 | `translateX` 4次 ±6px 抖动 | 400ms | ease |
| 选项 hover | `transform` scale(1.01) | 150ms | ease |
| 解析展开 | `max-height` 0→auto + `opacity` | 300ms | ease-out |

### 3.8 响应式断点

| 断点 | 布局变化 |
|------|---------|
| >768px | 卡片最大宽度 640px，居中，工具栏顶部横排 |
| ≤768px | 卡片占满宽 92%，工具栏折叠为底部浮层 |
| ≤480px | 字体缩小 1px，选项按钮全宽竖排 |

### 4.4 填空题答案匹配规则

```javascript
function matchAnswer(userInput, correctAnswer) {
  // 1. 去首尾空格
  // 2. 支持 | 分隔的多个正确答案
  // 3. 忽略大小写
  // 4. 标准化多余空格
  const input = userInput.trim().replace(/\s+/g, ' ').toLowerCase()
  const answers = correctAnswer.split('|').map(a =>
    a.trim().replace(/\s+/g, ' ').toLowerCase()
  )
  return answers.includes(input)
}
```

### 5.4 Service Worker（离线缓存）

内联注册一个最小化 SW，缓存 index.html 自身：

```javascript
if ('serviceWorker' in navigator) {
  // 使用 Blob URL 内联 Service Worker
  const swCode = `
    self.addEventListener('install', e => {
      self.skipWaiting()
      caches.open('ds-quiz-v1').then(c => c.addAll(['./']))
    })
    self.addEventListener('fetch', e => {
      e.respondWith(caches.match(e.request).then(r => r || fetch(e.request)))
    })
  `
  const blob = new Blob([swCode], { type: 'application/javascript' })
  navigator.serviceWorker.register(URL.createObjectURL(blob))
}
```

## 6. 数据方案

### 6.1 题目数据结构

```typescript
interface Question {
  id: number           // 唯一编号，如 101 = 第1章第1题
  chapter: number       // 章节号 1-9
  chapterName: string   // "绪论"、"线性表" 等
  type: 'single' | 'fill'  // 选择题 / 填空题
  question: string      // 题目正文
  options?: string[]    // 选项数组（选择题专有），如 ["A. ...", "B. ...", ...]
  answer: string         // 正确答案："A" | "B" | "C" | "D" | "填空文本|备选答案"
  explanation: string   // AI 生成的解析
}
```

### 6.2 OCR 数据提取流程

```
PDF (40页扫描图)
  │
  ├── 步骤1：提取全部 40 页为 JPEG 图片
  │
  ├── 步骤2：OCR 识别每页文字
  │     ├── 选择题页：识别"题号 题干 A... B... C... D..."
  │     ├── 填空题页：识别"题号 题干（含下划线空白）"
  │     └── 答案页：识别"章节 题号 答案"
  │
  ├── 步骤3：结构化解析
  │     ├── 正则匹配题号、题干、选项
  │     ├── 答案与题目配对
  │     └── 填空题答案中的下划线替换为标准空白标记
  │
  ├── 步骤4：数据校验
  │     ├── 核对各章节题目数量与目录表是否一致
  │     └── 标记可疑条目供人工复核
  │
  └── 步骤5：生成解析
        └── AI 为每道题生成知识点解析
```

### 6.3 数据存储位置

题库 JSON 直接内嵌在 `<script>` 标签中，不单独加载文件。预计大小约 150-250KB（gzip 后约 50-80KB），对首屏加载影响极小。

## 8. 部署方案

### 8.1 GitHub Pages 部署步骤

```bash
# 1. 创建仓库
git init
git add index.html
git commit -m "数据结构刷题 v1.0"

# 2. 推送到 GitHub
git remote add origin git@github.com:<用户名>/ds-quiz.git
git branch -M main
git push -u origin main

# 3. 在 GitHub 仓库 Settings → Pages → Source: main branch → Save

# 4. 访问 https://<用户名>.github.io/ds-quiz/
```
